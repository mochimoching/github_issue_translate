## 内容

RPCスタイルのリモーティングサポートのほとんどが数年間公式に、または事実上非推奨となっているため、6.0 M1で即座に削除します(これにより、いくつかのモジュールのサブパッケージの過負荷も少し軽減されます)。

### コメント 3 by knoobie

**作成日**: 2023-04-05

@snicoll さん、コメントありがとうございます! RPCスタイルのリモーティングの非推奨化と削除については認識しています(それには全く問題ありません)。ただ、このコミットがJAX-WS「統合」も削除したことに気づきました。これは非推奨ではなく、まだ広く使用されていると思います。例えば、CXF 4/Jakarta 9+への移行で、この変更に至ったオープンissueがあります: https://issues.apache.org/jira/browse/CXF-8666

現在の回避策は、`CommonAnnotationBeanPostProcessor`をカスタマイズして、削除された行を再導入することです。

```java
@Configuration
public class WorkaroundForCxfConfig {

  @Autowired
  private CommonAnnotationBeanPostProcessor processor;

  @PostConstruct
  public void initialize() {
    processor.ignoreResourceType("jakarta.xml.ws.WebServiceContext");
  }
}
```
