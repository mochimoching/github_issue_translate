# ServletContainerInitializerのファイル名を古い`javax`プレフィックスから`jakarta`に更新

## 内容

**影響バージョン:** 6.0.0-SNAPSHOT

`WebApplicationInitializer`インターフェースを実装しても、実装が呼び出されることはありません。

これは[このファイル](https://github.com/spring-projects/spring-framework/blob/main/spring-web/src/main/resources/META-INF/services/javax.servlet.ServletContainerInitializer)がまだ古い`javax`名前空間で命名されているために発生しています。

回避策として、`jakarta.servlet.ServletContainerInitializer`のファイルをアプリに追加しました。

この問題は、[カスタム`WebApplicationInitializer`実装](https://github.com/spring-projects/spring-security/blob/e398fbf2a7585c745a6a8d9d4ae6e980dff33462/web/src/main/java/org/springframework/security/web/context/AbstractSecurityWebApplicationInitializer.java#L74)を使用するSpring Securityのサンプルで作業中に発見しました。
