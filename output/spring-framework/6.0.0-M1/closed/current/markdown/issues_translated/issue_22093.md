Spring Context Supportは_javax.mail:javax.mail-api_と_javax.activation:javax.activation-api_を使用しています。現在は_com.sun.mail:jakarta.mail_と_com.sun.activation:jakarta.activation_が利用可能です。

**[Juergen Hoeller](https://jira.spring.io/secure/ViewProfile.jspa?name=juergen.hoeller)** がコメント:

一般的に、私たちは最新版ではなく、ベースラインAPIに対してコンパイルしています...ただし、Servlet 4.0のように新しいインターフェースへの(オプションの)依存関係が必要な場合は例外です。Servlet 4.0の場合、4.0に対してコンパイルしながら、実行時には3.1との互換性を保持しています。この観点から、当面は共通のEE 7/8 APIアーティファクトをベースラインとして維持する予定です。

Jakarta提供のアーティファクトへの切り替えを希望される具体的な理由はありますか? 実際のアプリケーションプロジェクトでは、Oracle提供のアーティファクトを強制していないため、特にオプションの依存関係については、Jakartaバリアントを実行時に使用できます。単にコンパイル目的で一般的な参照APIとして使用しているだけです。
