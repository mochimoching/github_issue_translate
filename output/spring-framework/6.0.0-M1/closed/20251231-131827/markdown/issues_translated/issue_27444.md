# Jakarta EE 9アノテーションに加えて、レガシーJSR-250 `javax.annotation.PostConstruct`/`PreDestroy`とJSR-330 `javax.inject.Inject`のサポートを維持

**Issue番号**: [#27444](https://github.com/spring-projects/spring-framework/issues/27444)

**状態**: closed | **作成者**: mp911de | **作成日**: 2021-09-21

**ラベル**: in: core, type: enhancement

**URL**: https://github.com/spring-projects/spring-framework/issues/27444

**関連リンク**:
- Commits:
  - [774583d](https://github.com/spring-projects/spring-framework/commit/774583dfa7dba5c3440c4a1341809ab2f3a88780)

## 内容

Jakarta EE 9への移行後、`javax.annotation.PostConstruct`や`javax.inject.Inject`がテストでカバーされていない場合に機能しなくなることを見落としやすいです。これらのアノテーションがJakarta EE 9への移行後もクラスパスに残っている場合、簡単に発生するバグになる可能性があります。

少なくとも猶予期間の間、両方のアノテーションバリアントをサポートするか、これらのアノテーションが使用されている場合に早期に失敗するようにすると良いでしょう。
