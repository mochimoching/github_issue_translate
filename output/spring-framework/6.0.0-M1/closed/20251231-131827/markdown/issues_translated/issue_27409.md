## 内容

spring.ioブログで発表されたように、[Spring Framework 6.0はJDK17をベースラインとして要求します](https://spring.io/blog/2021/09/02/a-java-17-and-jakarta-ee-9-baseline-for-spring-framework-6)。

その結果、デフォルトのビルドパイプラインはJDK17を使用し(将来的にはJDK18+のバリアントも)、公開されるアーティファクトはJava 17言語レベルを使用する必要があります。

このタスクでは、現在のCIパイプラインを以下のようにリファクタリングします:
* CIコンテナイメージに17以降のJDKのみを含める
* メインおよびテストソースのコンパイルにJava 17言語レベルを一貫して使用する
* 現時点では、KotlinはJava 17を公式にサポートしていないため、当面はJDK 11に移行します
