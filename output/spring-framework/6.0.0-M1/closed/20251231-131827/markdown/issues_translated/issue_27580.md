## 内容

`MediaType`オブジェクトの順序付け方法を再検討する必要があります。現在、優先度順にメディアタイプをソートするために`Comparator`を使用していますが、これらは推移的である必要があります([#27488](https://github.com/spring-projects/spring-framework/issues/27488)を参照)。6.0では他の順序付けメカニズムを調査し、comparatorを完全に廃止して、より制限の少ない(ただし遅い可能性がある)順序付けアルゴリズムを採用することを検討するべきです。
