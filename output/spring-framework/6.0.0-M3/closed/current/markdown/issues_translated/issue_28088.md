## 内容

コンテキストがビルド時に事前処理され、それに対して最適化されたコードを生成する場合、Bean定義の登録は明らかな部分の1つですが、唯一のものではありません。コアコンテナには、他のコンポーネントが実装してコンテキストのセットアップに貢献できる抽象化とAPIが必要です。

一例は、`@EventListener`アノテーション付きメソッドをスキャンし、必要に応じてイベントリスナーを登録するものです。これらのコンポーネントは実際に何か他のものを置き換えるため、置き換えているBean定義を除外することをオプトインできます。
