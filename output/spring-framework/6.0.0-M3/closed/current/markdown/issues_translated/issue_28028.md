## 内容

Spring Nativeでのプロトタイプにより、複数のステートメントやコードブロックを扱うと反復的なコードになることが示されました。また、JavaPoetはコードスニペット上のインポートを解決しないため、コードアサーションが少し厄介になります。

この課題は、これらのユーティリティを`spring-core`に移植し、それを活用できるようにすることについてです。
