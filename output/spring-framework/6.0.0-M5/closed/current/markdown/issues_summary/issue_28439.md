## 課題概要

Spring WebFlux向けに、Spring MVCの`ResponseEntityExceptionHandler`に相当する基底クラスを追加する機能強化要求です。

**背景説明**:
- **ResponseEntityExceptionHandler**: Spring MVCで提供されている、フレームワークの標準例外を一元的に処理するための基底クラス
- **WebFlux**: Spring Frameworkのリアクティブ（ノンブロッキング）なWebフレームワーク
- **@ControllerAdvice**: コントローラー横断的な例外ハンドリングを定義するためのアノテーション

**課題の目的**:
Spring MVCと同様に、WebFluxでもフレームワークが発生させるすべての標準例外のエラーメッセージを簡単にカスタマイズできるようにすること。特にRFC 7807（Problem Details）形式でのエラーレスポンス生成をサポートします。

## 原因

WebFluxには、Spring MVCの`ResponseEntityExceptionHandler`に相当する統一的な例外ハンドリング基底クラスが存在しませんでした。

これにより：
- フレームワーク例外ごとに個別に`@ExceptionHandler`を実装する必要があった
- RFC 7807サポート（[#27052](https://github.com/spring-projects/spring-framework/issues/27052)）を利用する際の一貫性が欠けていた
- MVCとWebFluxで例外ハンドリングのアプローチが異なっていた

## 対応方針

[b64835d](https://github.com/spring-projects/spring-framework/commit/b64835d2c8d6e195aa6d553fda6230b392b6f4f0)のコミットで実装されました。

**実装内容**:
WebFlux版の`ResponseEntityExceptionHandler`基底クラスを新規追加：

1. **統一的な例外ハンドリング**: すべてのSpring WebFlux標準例外を一箇所で処理
2. **RFC 7807サポート**: Problem Details形式でのエラーレスポンス生成
3. **カスタマイズ容易性**: 基底クラスを継承して必要なメソッドをオーバーライドするだけで、エラーメッセージをカスタマイズ可能
4. **MVCとの一貫性**: Spring MVCの`ResponseEntityExceptionHandler`と同様のAPI設計

使用方法:
```java
@ControllerAdvice
public class CustomExceptionHandler extends ResponseEntityExceptionHandler {
    // 必要に応じてメソッドをオーバーライドして
    // エラーレスポンスをカスタマイズ
}
```
