## 課題概要

`UrlResource`のインスタンスを作成する際に、チェック例外（`IOException`）ではなく非チェック例外（`UncheckedIOException`）をスローするファクトリーメソッドを追加する機能強化です。

**背景説明**:
- **UrlResource**: Spring FrameworkのResourceインターフェース実装で、URL経由でリソースにアクセスするクラス
- **チェック例外**: try-catchまたはthrowsでの明示的な処理が必要な例外（例: `IOException`）
- **非チェック例外**: 処理が強制されない例外（例: `RuntimeException`やその派生）
- **UncheckedIOException**: `IOException`をラップした非チェック例外

**問題の詳細**:
現在の`UrlResource`コンストラクタは`MalformedURLException`などのチェック例外をスローします：

```java
// 現状: チェック例外が必要
try {
    UrlResource resource = new UrlResource("https://example.com");
} catch (MalformedURLException e) {
    // 処理
}
```

これにより、Stream APIやOptional APIでの使用が困難：
```java
// コンパイルエラー: チェック例外を処理できない
Stream.of("url1", "url2")
      .map(UrlResource::new)  // ❌ 使用不可
      .collect(Collectors.toList());
```

## 原因

`UrlResource`の設計が、チェック例外をスローするコンストラクタのみを提供していたため。

Java 8以降の関数型APIやラムダ式では、チェック例外を扱うことが困難で、以下のような用途で不便：
- Stream API での変換処理
- Optional の map/flatMap 操作
- CompletableFuture のチェーン処理
- その他の関数型プログラミングパターン

## 対応方針

[f07e7ab](https://github.com/spring-projects/spring-framework/commit/f07e7ab39d04c0f8b27f3e5678227952cc3ac380)のコミットで実装されました。

**実装内容**:

非チェック例外をスローするファクトリーメソッドを追加：

```java
public class UrlResource {
    // 既存のコンストラクタ (チェック例外)
    public UrlResource(String path) throws MalformedURLException { ... }
    public UrlResource(URI uri) throws MalformedURLException { ... }
    
    // 新規ファクトリーメソッド (非チェック例外)
    public static UrlResource from(String path) 
        throws UncheckedIOException { ... }
    public static UrlResource from(URI uri) 
        throws UncheckedIOException { ... }
}
```

**使用例**:
```java
// Stream APIでの使用
List<UrlResource> resources = Stream.of("url1", "url2", "url3")
    .map(UrlResource::from)  // ✅ 使用可能
    .collect(Collectors.toList());

// Optionalでの使用
Optional<String> urlOpt = ...;
UrlResource resource = urlOpt.map(UrlResource::from)
                              .orElseThrow();
```
