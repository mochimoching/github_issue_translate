## 内容

新しい`RuntimeHints` APIは、実行時のリフレクション/プロキシ/リソースのニーズに関するヒントを記述することができます。

問題は、実行時の単一の呼び出しが複数の異なるヒントでカバーできることです。

例えば、クラス`MyClass`の`myMethod`メソッドに対するリフレクションイントロスペクションは、次のいずれかでカバーできます：

* `myMethod`がpublicの場合は`MemberCategory.INTROSPECT_PUBLIC_METHODS`
* `myMethod`がpublicの場合は`MemberCategory.INVOKE_PUBLIC_METHODS`
* すべての場合は`MemberCategory.INTROSPECT_DECLARED_METHODS`
* すべての場合は`MemberCategory.INVOKE_DECLARED_METHODS`
* `MyClass::myMethod`の特定のイントロスペクションまたは呼び出しリフレクションエントリ

登録されたヒントが特定のユースケースに十分かどうかをチェックするために必要な知識があるため、テストおよび#27981で使用するための`Predicate<RuntimeHints>`静的ユーティリティを導入すべきです。
