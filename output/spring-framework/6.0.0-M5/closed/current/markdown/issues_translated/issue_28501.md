## 内容

## 概要

`UrlResource`のコンストラクタはチェック例外をスローするため、`java.util.Stream`や`java.util.Optional` APIで使用したり、チェックされた`IOException`が望ましくない他のシナリオで使用することが困難です。

## 提案

このようなユースケースをサポートするために、`UncheckedIOException`をスローする`UrlResource`のファクトリーメソッドを導入すべきです。最初は、`URI`または`String`を受け入れるコンストラクタのバリアント用に導入します。
