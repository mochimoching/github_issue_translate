## 内容

現在、`ApplicationContextAotGenerator`は常に`__.BeanFactoryRegistrations`クラスを生成します。コンテキスト階層サポートのために複数の登録クラスが必要になるため、異なる名前を使用すべきです。また、`__`以外のパッケージを使用できるように、ターゲットクラスを提供できるようにしたいと考えています。
